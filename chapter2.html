01

Lists

A list is a single box that can hold many items in order — like a shopping list, but for your code.

fruits = ["apple", "banana", "mango"] numbers = [10, 20, 30, 40]
  • Written with square brackets [ ], items separated by commas
  • Can hold mixed types: ["Rahul", 20, True]
  • Changeable — you can add, remove, or edit items anytime
  • Keeps the order you put things in
i

Think of a list like a row of numbered lockers. Each locker holds one item, and the locker numbers never change — that's exactly how "indexing" works, coming up next.

02

Accessing elements from a list

Every item in a list has a position number, called an index — and counting always starts from 0, not 1.

fruits = ["apple", "banana", "mango"] print(fruits[0]) # apple — first item print(fruits[1]) # banana — second item print(fruits[2]) # mango — third item
i

It's like locker numbers starting from 0 instead of 1. Locker 0 is the first locker, locker 1 is the second — a bit unusual at first, but you'll get used to it fast.

03

Negative indexing

Python also lets you count backward from the end of the list, using negative numbers.

fruits = ["apple", "banana", "mango"] print(fruits[-1]) # mango — last item print(fruits[-2]) # banana — second-last item
i

Think of -1 as "the last one," -2 as "second from the end," and so on — like counting steps backward from the end of a queue instead of walking all the way from the front.

04

Adding elements to a list

Lists are changeable, so you can add new items whenever you want.

fruits = ["apple", "banana"] fruits.append("mango") # adds to the end # fruits is now: ["apple", "banana", "mango"] fruits.insert(1, "grape") # adds at a specific position # fruits is now: ["apple", "grape", "banana", "mango"] fruits.extend(["kiwi", "pear"]) # adds multiple items at once
MethodWhat it does
append(x)Adds one item to the end
insert(i, x)Adds one item at position i
extend(list)Adds every item from another list
05

Deleting elements from a list

Just as easily as you add, you can remove items too.

fruits = ["apple", "banana", "mango"] fruits.remove("banana") # removes by NAME (first match) fruits.pop() # removes the LAST item, and returns it fruits.pop(0) # removes item at position 0 del fruits[0] # removes item at that position too fruits.clear() # empties the whole list
i

remove() says "get rid of this item by name." pop() says "take out the item at this position (and hand it to me)." clear() says "empty the whole box."

06

Basic list operations

A few simple, everyday things you'll do with lists all the time.

a = [1, 2, 3] b = [4, 5] print(a + b) # [1, 2, 3, 4, 5] — joins two lists print(a * 2) # [1, 2, 3, 1, 2, 3] — repeats the list print(len(a)) # 3 — how many items print(2 in a) # True — is 2 present in the list? print(a[0:2]) # [1, 2] — slicing: items from index 0 up to (not including) 2
i

Slicing is like cutting a piece from a loaf of bread — a[0:2] means "cut from position 0 up to, but not including, position 2."

07

Built-in list methods

Python gives lists several ready-made tools ("methods") so you don't have to build these yourself.

nums = [4, 1, 3, 1, 2] nums.sort() # [1, 1, 2, 3, 4] — sorts in place nums.reverse() # flips the current order nums.count(1) # 2 — how many times 1 appears nums.index(3) # position where 3 first appears new_list = nums.copy() # makes a separate copy
MethodWhat it does
sort()Arranges items in order (small → big)
reverse()Flips the current order
count(x)Counts how many times x appears
index(x)Finds the position of x
copy()Makes an independent duplicate
08

List comprehension

A short, elegant way to build a new list from an existing one, in a single line.

# the long way squares = [] for x in range(5): squares.append(x * x) # the short way — list comprehension squares = [x * x for x in range(5)] # [0, 1, 4, 9, 16] # with a condition evens = [x for x in range(10) if x % 2 == 0] # [0, 2, 4, 6, 8]
i

Read it like a sentence: "give me x * x, for every x in range(5)." Same result as the loop above, just packed into one clean line.

09

Functions

A function is a reusable, named block of instructions — write it once, use it as many times as you like.

def greet(name): print("Hello, " + name) greet("Rahul") # Hello, Rahul greet("Priya") # Hello, Priya
  • def starts a function definition
  • name here is a parameter — a placeholder for whatever value you pass in
  • Calling greet("Rahul") actually runs the code inside
def add(a, b): return a + b result = add(3, 5) print(result) # 8
i

A function is like a recipe card: you write the steps once ("how to make tea"), and every time you want tea, you just follow the card again — no need to re-write the steps.

10

Namespace

A namespace is simply "where a name lives" — it decides which variables are visible from where.

x = "global" # lives OUTSIDE any function def show(): y = "local" # only exists INSIDE this function print(x) # can still read the global one print(y) show() print(x) # works — x is global print(y) # ❌ error — y doesn't exist out here
NamespaceWhere it lives
LocalInside a function only — disappears once the function finishes
GlobalOutside all functions — visible everywhere in the file
i

Think of a function like a private room. Things you create inside that room (local variables) stay in the room and vanish once you leave. Things from the main house (global variables) can still be seen from inside the room.

11

Returning multiple values

A Python function isn't limited to sending back just one value — it can hand back several at once, separated by commas.

def get_name_and_age(): name = "Rahul" age = 20 return name, age n, a = get_name_and_age() # unpacked into two variables print(n) # Rahul print(a) # 20
i

It's like a vending machine that hands you a snack AND a drink in one go — you just take both out of the bag separately when you get home. Behind the scenes, Python actually bundles them into a "tuple," then n, a = unpacks it back into two named boxes.
